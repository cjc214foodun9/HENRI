### **1\. Epiplexity Theory and Computationally Bounded Information**

Classical information frameworks—such as Shannon entropy $H(X)$ and Kolmogorov complexity $K(x)$—assume observers with unbounded computational capacity. Under these classical models:

1. Deterministic transformations cannot increase information ($H(f(X)) \\le H(X)$).  
2. Information is invariant to factorization order ($H(X,Y) \= H(Y,X)$).  
3. Likelihood maximization is merely distribution matching.

For a computationally bounded observer (restricted to runtime budget $T$), information bifurcates into two distinct physical components:

$$MDL\_T(X) \= S\_T(X) \+ H\_T(X)$$  
Where:

* **Time-Bounded Entropy (**$H\_T(X)$**):** The residual, inherently unpredictable random noise content under runtime constraint $T$.  
* **Epiplexity (**$S\_T(X)$**):** The structural, learnable information content extracted into system parameters/weights that minimizes the two-part description length under budget $T$:

$$P^\* \= \\arg\\min\_{P \\in \\mathcal{P}\_T} \\left\\{ \\vert{}P\\vert{} \+ \\mathbb{E}\\left\[\\log \\frac{1}{P(X)}\\right\] \\right\\}$$  
$$S\_T(X) \= \\vert{}P^\*\\vert{}, \\quad H\_T(X) \= \\mathbb{E}\\left\[\\log \\frac{1}{P^\*(X)}\\right\]$$

### **2\. Resolution of Information Paradoxes within Project HENRI**

#### **Paradox 1: Creation of Information via Deterministic Processes**

In classical information theory, synthetic data or deterministic simulation yields zero new mutual information. For a time-bounded observer $T$, applying a non-linear transformation $f$ whose inverse $f^{-1}$ lacks a short, efficiently executable program dramatically increases epiplexity:

$$S\_{T'}(f(X)) \\gg S\_T(X)$$

Under one-way function constraints, factorizing data in reverse computational order forces higher time-bounded entropy and forces the system to learn richer structural representations:

$$H\_{\\text{Poly}}(X\\vert{}Y) \+ H\_{\\text{Poly}}(Y) \> H\_{\\text{Poly}}(Y\\vert{}X) \+ H\_{\\text{Poly}}(X) \+ \\omega(\\log n)$$  
By formatting sensory feeds and task specifications in reverse computational order (e.g., target outcome state followed by intermediate action trajectories), HENRI's Kuramoto swarm builds deeper phase manifolds, maximizing out-of-distribution (OOD) transfer capability.

Maximum likelihood modeling under compute bound $T$ forces the emergence of predictive sub-circuits that do not explicitly exist in the generative code.

### **1\. Ambient Dimension Mismatch ($\\mathbb{C}^{65536}$) and RMS Normalization**

#### **The Diagnostic Failure Mode (Phase 2.6)**

In high-dimensional complex Hilbert spaces $\\mathcal{H}^d$ with ambient dimension $d\_{\\text{ambient}} \= 65536$, the un-squared $L\_2$ norm of an off-manifold residual vector scales as $\\mathcal{O}(\\sqrt{d\_{\\text{ambient}}})$.

When evaluating candidate state predictions $\\mathbf{\\Psi}\_{\\text{pred}}$ against an Extended Dynamic Mode Decomposition (EDMD) invariant projection matrix $P\_{\\text{inv}}$:

$$\\text{residual}\_{\\text{raw}} \= \\left\\Vert{} \\mathbf{\\Psi}\_{\\text{pred}} \- \\mathbf{\\Psi}\_{\\text{pred}} P\_{\\text{inv}} \\right\\Vert{}\_2$$  
Small orthogonal components across $d\_{\\text{ambient}} \= 65536$ dimensions accumulated, yielding raw penalty norms of $\\approx 80.0$ to $90.42$. When evaluated against the rejection threshold $\\epsilon\_{\\text{reject}} \= 0.35$, **93% to 100% of all candidate predictions were classified as off-manifold and rejected**.

This triggered the fallback selection path:

$$\\text{ranked} \= \\text{admissible if admissible else results}$$  
The system fell back to unconstrained EFE ranking while simultaneously injecting an additive scalar penalty of $\\lambda\_{\\text{active}} \\cdot \\text{residual}\_{\\text{raw}} \\approx 1.4162 \\times 90.4 \\approx 128.0$ into the pragmatic cost. This caused EFE values on environment cd82 to explode from $\\sim \+41.0$ to $\\sim \+113.0$, introducing complete trajectory instability.

#### **Mathematical Resolution: Dimension-Independent RMS Residual**

To ensure the penalty functional measures fractional displacement off the invariant manifold rather than ambient vector size, the residual norm is normalized by $\\sqrt{d\_{\\text{ambient}}}$:

$$\\text{residual}\_{\\text{RMS}} \= \\frac{\\left\\Vert{} \\mathbf{\\Psi}\_{\\text{pred}} \- \\mathbf{\\Psi}\_{\\text{pred}} P\_{\\text{inv}} \\right\\Vert{}\_2}{\\sqrt{d\_{\\text{ambient}}}}$$  
For $d\_{\\text{ambient}} \= 65536$, $\\sqrt{d\_{\\text{ambient}}} \= 256.0$. A raw $L\_2$ norm of $80.0$ scales to an RMS residual of:

$$\\text{residual}\_{\\text{RMS}} \= \\frac{80.0}{256.0} \= 0.3125$$  
This rescales the residual into the physical domain $\[0.0, 1.0\]$, restoring proper interaction with the rejection threshold $\\epsilon\_{\\text{reject}} \= 0.35$ and guaranteeing non-empty admissible candidate sets.

import torch

def score\_actions(self, current\_state: torch.Tensor, candidate\_actions: torch.Tensor) \-\> dict:

    """

    Score candidate actions using dimension-independent RMS manifold residuals,

    PEARL training-free reasoning unit repair, active MACURA constraint scaling,

    and preference resonance.

    """

    \# 1\. Project forward latent states using Wave-JEPA transition core

    candidate\_predictions \= self.transition\_network(current\_state, candidate\_actions)

    

    \# 2\. Compute dimension-independent RMS residual

    d\_ambient \= candidate\_predictions.shape\[-1\]

    sqrt\_d \= float(torch.sqrt(torch.tensor(d\_ambient, dtype=torch.float32, device=candidate\_predictions.device)))

    

    \# Raw L2 projection error against EDMD projection matrix P\_inv

    raw\_residuals \= torch.norm(

        candidate\_predictions \- torch.matmul(candidate\_predictions, self.P\_inv),

        p=2, dim=-1

    )

    

    \# Scale to dimension-independent RMS domain

    rms\_residuals \= raw\_residuals / sqrt\_d

    

    \# 3\. PEARL Unit Materialization & Anchor Identification

    \# Evaluate reasoning units: retain valid anchors, isolate rejected units

    admissible\_indices \= \[

        idx for idx, res in enumerate(rms\_residuals) 

        if res.item() \<= self.constraint\_reject\_thresh

    \]

    

    repaired\_candidates \= candidate\_predictions.clone()

    pearl\_repair\_cost \= torch.zeros\_like(rms\_residuals)

    repaired\_flags \= \[False\] \* candidate\_predictions.shape\[0\]

    

    \# 4\. PEARL Judge-Feedback Local Repair Loop

    \# If a candidate is rejected, perform targeted unit phase repair using Zone C preferences

    for idx in range(candidate\_predictions.shape\[0\]):

        if idx not in admissible\_indices and hasattr(self, "zone\_c\_preferences") and self.zone\_c\_preferences is not None:

            \# Extract preference guidance vector from Zone C

            pref\_vector \= self.zone\_c\_preferences.get\_dominant\_attractor()

            if pref\_vector is not None:

                \# Perform localized phase blend over corrupted sub-dimensions

                repaired\_state \= (1.0 \- self.pearl\_repair\_alpha) \* candidate\_predictions\[idx\] \+ self.pearl\_repair\_alpha \* pref\_vector

                repaired\_state \= repaired\_state / torch.norm(repaired\_state, p=2, dim=-1, keepdim=True)

                

                \# Re-evaluate RMS residual post-repair

                repaired\_raw \= torch.norm(repaired\_state \- torch.matmul(repaired\_state, self.P\_inv), p=2, dim=-1)

                repaired\_rms \= repaired\_raw / sqrt\_d

                

                if repaired\_rms.item() \<= self.constraint\_reject\_thresh:

                    repaired\_candidates\[idx\] \= repaired\_state

                    rms\_residuals\[idx\] \= repaired\_rms

                    raw\_residuals\[idx\] \= repaired\_raw

                    pearl\_repair\_cost\[idx\] \= self.pearl\_edit\_penalty

                    repaired\_flags\[idx\] \= True

                    admissible\_indices.append(idx)

    \# 5\. Calculate MACURA dynamic lambda

    active\_lambda \= float(self.calculate\_macura\_lambda(self.loss\_ema))

    

    \# 6\. Compute EFE Pragmatic contribution with PEARL Repair Penalties

    base\_pragmatic \= self.compute\_base\_pragmatic(repaired\_candidates)

    preference\_resonance \= self.compute\_preference\_resonance(repaired\_candidates)

    

    efe\_scores \= base\_pragmatic \+ (active\_lambda \* rms\_residuals) \- (self.beta\_pragmatic \* preference\_resonance) \+ pearl\_repair\_cost

    

    \# 7\. Apply hard-rejection filtering with audit fallback telemetry

    if len(admissible\_indices) \> 0:

        admissible\_scores \= efe\_scores\[admissible\_indices\]

        winning\_sub\_idx \= int(torch.argmin(admissible\_scores).item())

        winning\_idx \= admissible\_indices\[winning\_sub\_idx\]

        fallback\_executed \= False

        residual\_type \= "ACCEPTED\_CLEAN" if not repaired\_flags\[winning\_idx\] else "ACCEPTED\_PEARL\_REPAIRED"

    else:

        \# Fallback to unconstrained argmin if all candidates exceed threshold post-repair

        winning\_idx \= int(torch.argmin(efe\_scores).item())

        fallback\_executed \= True

        residual\_type \= "RESIDUAL\_FAILED\_STRICT\_GATE"

        

    return {

        "best\_action\_idx": winning\_idx,

        "best\_efe": float(efe\_scores\[winning\_idx\].item()),

        "constraint\_penalty": float(rms\_residuals\[winning\_idx\].item()),

        "raw\_l2\_residual": float(raw\_residuals\[winning\_idx\].item()),

        "constraint\_rejected": bool(winning\_idx not in admissible\_indices),

        "fallback\_executed": fallback\_executed,

        "admissible\_count": len(admissible\_indices),

        "lambda\_active": active\_lambda,

        "preference\_resonance": float(preference\_resonance\[winning\_idx\].item()),

        "pearl\_repaired": repaired\_flags\[winning\_idx\],

        "residual\_type": residual\_type,

        "efe\_scores": efe\_scores.detach().cpu().tolist(),

        "predicted\_latents": repaired\_candidates\[winning\_idx\].detach()

    }
